Housing prices Copenhagen

Chapter 7 Open Street Map

The location information has been obtained from Open Street Map. We have selected services inside a buffer of approx. 2 km around the study area (Figure 7.1 and 7.2), assuming that people may use then (e.g. parks, parking areas) outside Copenhagen City but they are close enough to people houses. Then, we estimate the accessibility to each services (i.e. feature) in the centroid of the grid cell of 100m x 100m; i.e. Potential model chapter 1. We selected two types of features; i. areal features and ii. point features; where the mass of the service (\(M_j\)) is the area of the polygon in \(m^2\) or equal for all services (i.e. 1), respectively. On the other hand, the distance (\(d_{ij}\) in \(m^2\)) is the distance between the centroid of the polygon (i.e. areas features) or the point (i.e. point feature) and the centroid of the grid cells.

7.1 Features

7.1.1 Areal features

The selected features and their expected effect on housing prices are reported in Table 7.1.

Hospital, and Schools (e.g. schools) may create traffic and noise. Therefore, they may have a negative effect on housing prices (Gultekin and Yamamura (2006)).

osm_poly <- merge(osm_poly_list, osm_poly_dnld, by = c("key", "value")) %>% 
  # Convert to sf object
  st_sf() %>% 
  # Transform CRS
  st_transform("EPSG:25832") %>% 
  # Remove non-valid polygons (i.e. 2 polygons)
  mutate(valid = st_is_valid(.)) %>% 
  filter(!is.na(valid)) %>% 
  # From Meadow get only "Kalvebod Fælled" 
  filter(feature != "Meadow" |
         (key == "landuse" & value == "meadow" & name == "Kalvebod Fælled")) %>%
  mutate(feature = case_when(feature == "Meadow" ~ "Kalvebod_Fælled",
                             TRUE ~ feature)) %>% 
  # Mass of the service (area of the polygons)
  mutate(m_service = as.numeric(units::set_units(st_area(.), m^2))) %>% 
  # crop to Bounding of the study area %>% 
  st_crop(bbox_sf) %>% 
  # Nest by feature
  group_by(feature) %>% 
  nest() %>% 
  ungroup() %>% 
  rename(data_poly = data) %>% 
  # Get centroids  
  mutate(data_points = map(data_poly, st_centroid))

7.2 Potential model

# Aux. function for calculating the potential
f_pot <- function(df, fun, span, beta) {
  mcpotential(x = df,
              y = grids100m$data_points[[1]],
              var = "m_service",
              fun = fun,
              span = span,
              beta = beta,
              limit = 5 * span)
  }

# Estimate potential (in percentage - relatively to the maximum)
pot_est <- list()
for(i in seq_along(OMS$data_points)) {
  # Values
  potential <- f_pot(OMS$data_points[[i]],
                     OMS$fun[i], 
                     OMS$span[i],
                     OMS$beta[i])
  # Percentage relatively to the maximum
  pot_est[[i]] <- 100 * potential / max(potential)
  # Add names
  names(pot_est)[[i]] <- OMS$feature[i] 
}

# Add values to the grid cells of 100 x 100 m (grids100m)
pot <- rbind(pot_est) %>%
  as_tibble() %>% 
  unnest(everything())

grids100m <- grids100m %>% 
  bind_cols(pot %>% nest(pot = everything()))
